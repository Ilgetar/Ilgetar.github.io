# 人物介绍

模拟人生设计师 - 威尔 - 莱特

这一类型的大师 —— 即上帝游戏之神（the god of God games）—— 是威尔・莱特（Will Wright）。1989 年，《模拟城市》（SimCity）发布，该游戏的目标是设计并管理一个现代城市；2000 年，《模拟人生》（The Sims）横空出世，在游戏中，玩家需要在一个普通的城郊环境中照看一个家庭。莱特创作的模拟重新定义了游戏的边界。「我突然想到，大多数书籍和电影往往是关于现实情景的（realistic situations）」他说，「为什么游戏不能是这样呢？」对于游戏设计师来说，莱特是在游戏形式上的左拉：他将游戏核心关注的主题从神话、幻想和暴力转向平凡的社会生活。

# 零、游戏相关的基本概念

我们所见只是我们能见的，而游戏是一个全局视角。作为设计师，不仅要接触电子设备，还要处理玩家的想象力。

# 1. 游戏是什么？

有棋牌游戏，社交游戏，益智游戏等各种分支。但是游戏实际上就是玩耍的一个分支，是我们在现实中试错成本很高的一个小玩具。而游戏本身跨学科的性质也导致你需要联系各种方面去学习。

# 2. 跨学科的设计模式

你必须要首先成为一个设计师。你需要去看各种各样的学科知识，作为一个设计师，去学习，去融入创造。向别人学习，再向自己学习。你要持续不断的去产出一些小的设计游戏。而不是总是苦苦思考一个宏大的愿景。

# 3. 探索决策的分支

你需要去协调各种设计方案与决策的方向，包括预算，结构等信息。你可以从大的树根出发，进一步去细化小的枝叶。实际上你并不知道自己要去哪里，但是你到时候要知道，也不要过犹不及。

# 4. 拥抱约束

没有约束，就没有设计。比如技术、图像处理、信息、想象力的约束。你越是了解你的约束，你就越能突破。

# 5. 突破零和博弈

在零和博弈中，所有人分数总和为 0。那么就不会有两个赢家。不妨不要在游戏里面定义输赢，让玩家自己去定义结构和输赢。让游戏更开放，例如 “模拟游戏”。

# 6. 拓展玩家的创造力

设备是我们的延伸。玩家可以利用这个游戏的工具，去创造属于玩家自己的独特的东西，激发玩家的创造力。比如建立游戏社区。

# 一、游戏构思的产生

# 1. 找到你灵感的调色盘

作为一个游戏设计师，你应该保持学习。在不断的学习中，去寻找自己感兴趣的东西。例如说，在模拟城市的时候，会极大激发个人对于规划城市的兴趣。

# 2. 没有边界的探索

现在针对一个感兴趣的门类可以无限探索，越来越小的限制。最需要锻炼的反而是思想和创造力。和万物交互，去体验。

# 3. 模拟人生

对相关的数据进行研究。将相关的数据记录起来，也许哪一天就用上了呢？比如说一个老人和小孩，在对于最珍惜的东西之上是一样的这个信息。

# 4. 聚焦你的概念

需要做一个主题的游戏时候，我会先去更加深入和这个主题相关的资料。然后再去寻找特点，寻找有趣的地方，例如结构，背景等。并且最好有多个视角去思考这个问题。

# 5. 相信直觉

相信直觉能给你带到你该去的地方。也许在一开始，某些人会认为模拟城市是只有部分人喜欢的，但是不是的。

# 二、早期原型

有想法就去制作原型，使用最少的成本去发现有趣的部分

一个原型通常是交互的，没有局限的，例如可以使用纸张等小物品，它可以是任何形式的

制作可以触摸的原型

寻找乐趣，然后去延伸到游戏其他部分

使用最合适的平台

三，游戏和故事的关系

人们对于世界的认知来源于自己的感受到的事物，进而在脑海里构建属于自己的思维模型。后来进一步延伸到游戏和故事的方法，通过游戏体验一些抽象的概念，通过他人的的故事去理解新的事物。

游戏和故事本身就有教育意义，也是我们愿意消费的领域。

真正好的游戏可以产出故事，反之亦然。

我们通过共情感受到故事的情绪，而游戏的感染力来源于自己的行为，自己的所作所为产生对应的情感。进而言之，故事没有参与感，而游戏有参与感，两者完全不同。

让玩家自己来讲述故事，例如引擎电影。

四，心理与游戏

通过与玩家的心理打交道，构建一个目标和一个限制

一般来说，玩家要在游戏中越重要。

游戏中会存在一些与现实不符合的现象，但是那是玩家想看到的，例如核电站起火就爆炸。

构建玩家的社区，让玩家进行自我的构建。

鼓励你的玩家，但是别挡道。让玩家成为主人。

flow 是一种介于难也不难的状态，可以使用 DDA 动态调节游戏的难度，让玩家处于 FLow 状态，保持对游戏的兴趣。

失败和成功的网状闭环，你要让游戏循序渐进。尽量让游戏的失败变得有趣，而不是重复。

游戏成功不应该由作者决定，而是提供不同的成功路径给玩家。

比起奖励，探索和发现要更加吸引人。（彩蛋）

马斯洛需求理论

《大脑的地图》

五，游戏设计

在游戏中可以使用概率来解决很多关于计算的问题。比如说计算面积，可以使用随机数处理。

在处理信号和暗示的时候，你需要明确自己给的信号是有明确关联的，而不是一个随机的事情。比如说开了门就会有怪物出现这样，要有明确的提示。

玩家希望自己的命运是可掌握的，而不是因为一些无法控制的东西，例如莫名其妙的死亡。（参考第五人格的厄运震慑）

尽快去迭代你的问题，及时修复那些存在的问题，以免夜长梦多。

如果你进入了瓶颈，进入了局部最大值，请你解构已有，尝试突破。

开发时候优先选择成本小的开发，同时参考重要性。

面巾纸测试：寻找从没有接触过游戏的人来测试，并且以后再也不玩。

测试人员给出的解决方案不一定可靠，但是背后的问题一定真实需要解读

六，细节设计

音效音乐设计是非常重要的部分，可以考虑生成式音乐

配音可以架空使用抽象的胡言乱语（））

七，游戏宣传

对不同的人讲述不同的人他们关心的事情，比如记者关心独特的地方。

对玩家来说，你需要表达出玩家可以做什么，告诉玩家什么是有趣的。

在推销时候，注意不要说的太多，言多必失。留白让玩家自己去补充。

推销时候自己就要用有活力有兴趣。
