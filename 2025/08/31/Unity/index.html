# 二、Unity 初步认识

# 1. 什么是 unity

一套有完善体系和编辑器的游戏开发工具，带有可复用代码块。目前 unity 已经占有市场超过 50 份额。

Unity 是一款跨平台的游戏引擎，包含以下系统：渲染引擎、物理引擎、碰撞检测系统、音效引擎、脚本引擎、动画系统、人工智能、网络引擎、场景管理等。Unity 提供了丰富且强大的各类功能，使开发者可以将精力集中在游戏逻辑和设计上，从而简单快速的创建游戏。主要支持的平台有：Windows/IOS/Android/Web/Xbox/PS/Wii

# 2.unity 的特点

# 3. 基本开发逻辑

游戏构建开发逻辑：提出雏形玩法，玩法规则具体化，规划各功能模块，落实到相关数据结构和算法实现。

游戏制作流程：将美术素材和逻辑功能整合创造角色，摆放好角色，创建场景，创建好所有需要的场景。

游戏的重要性是最重要的。

# 4.unity 重要概念

# 游戏对象

游戏中出现的物体，包括游戏角色，特效，背景模型，摄像机（决定场景的显示范围和显示角度），灯光（决定 3d 模型的色泽），管理对象（用于管理角色生成等）等

# 组件

将 “响应操作”“播放动画” 等功能与美术素材相结合的载体。需要什么功能才添加什么组件。脚本属于可以定制的组件。网格渲染器组件可以将 3D 模型资源显示。

# 资源

3D 模型与 2D 图片等数据

# 场景

流程：以画面作用目的为基础将游戏过程切分为不同的环节，称为流程

段落：用于切分流程的画面，例如 “游戏画面”“结果画面”

场景：游戏中一个个场面或者段落的组成单元，可以看成一个用于容纳游戏对象的箱子

场景：容纳游戏对象的箱子；场景用于一个画面的创建

# 预设

预设用于复制游戏对象。可以在游戏运行时候通过脚本来创建游戏对象。即它支持运行时生成。特性：源对象的修改可以直接反映到复制生成的游戏对象上。反之亦然。

预设的优势：使得复制后的修改方便，可以简单的复制到任何场景。
