# 一、网页制作基本概述

# 1. 基本概念

  • HTML

    HTML 的全称为超文本标记语言，是一种标记语言。它包括一系列标签，通过这些标签可以将网络上的文档格式统一，使分散的 Internet 资源连接为一个逻辑整体。HTML 文本是由 HTML 命令组成的描述性文本，HTML 命令可以说明文字，图形、动画、声音、表格、链接等。HTML 文档属于纯文本文档（ASCII），意味着不包含与平台或程序有关的信息。

    超文本是一种组织信息的方式，它通过超级链接方法将文本中的文字、图表与其他信息媒体相关联。这些相互关联的信息媒体可能在同一文本中，也可能是其他文件，或是地理位置相距遥远的某台计算机上的文件。这种组织信息方式将分布在不同位置的信息资源用随机方式进行连接，为人们查找，检索信息提供方便。

    HTML 提供给不同用户相同的效果展示。广义的 HTML5 包括了 HTML，CSS 和 JavaScript 在内的一套技术组合。

    注意！HTML 对于排版设计没有一字有提，只负责内容的结构式划分。

  • CSS

    层叠样式表 (英文全称：Cascading Style Sheets) 是一种用来表现 HTML（标准通用标记语言的一个应用）或 XML（标准通用标记语言的一个子集）等文件样式的计算机语言。CSS 不仅可以静态地修饰网页，还可以配合各种脚本语言动态地对网页各元素进行格式化。

    CSS 能够对网页中元素位置的排版进行像素级精确控制，支持几乎所有的字体字号样式，拥有对网页对象和模型样式编辑的能力。

    即用于美化 HTML 网页。

  • 核心理念

    • 一定要让网页在绝大多数浏览器中都能正常显示。

    • 一定要让内容结构良好而且清晰，便于阅读理解。

  • 三级网站页面结构

    一级页面也就是网站的首页，二级页面是链接到首页的，从首页点击进去的就是二级；三级页面链接到二级页面，从二级点击进去的就是三级。三级网站页面也可以叫内容详情页。\n\n 频道就是该网站内容的分类，往往在首页会有标题，比如” 娱乐 “、” 音乐 “、” 新闻 “等等。\n\n 所谓网站（Website），就是指在网际网路（因特网）上，根据一定的规则，使用 HTML 等工具制作的用於展示特定内容的相关网页的集合。简单地说，网站是一种通讯工具，就像布告栏一样，人们可以通过网站来发布自己想要公开的资讯（信息），或者利用网站来提供相关的网路服务（网络服务）。人们可以通过网页浏览器来访问网站，获取自己需要的资讯（信息）或者享受网路服务。\n\n 网页设计一般分为三种大类：功能型网页设计（服务网站；B/S 软件用户端）、形象型网页设计（品牌形象站）、信息型网页设计（门户站）。设计网页的目的不同，应选择不同的网页策划与设计方案。\n\n 在你眼前，出现在显示器上的这个 “东西”，就是一个网页。网页实际是一个页面文件，它存放在世界某个角落的的某一台计算机中，而这台计算机必须是与互联网相连的。网页经由网址（URL）来识别与存取，当我们在浏览器输入网址后，经过一段复杂而又快速的程序。网页文件会被传送到你的计算机，然后再通过浏览器解释网页的内容，再展示到你的眼前。\n\n 综上所述，三级网站页面指的一个网站的详情页面。

  • URL

    • 统一资源定位符。主要有 HTTP（超文本传输协议），FTP（文件传输协议），mailto 等类型；

    • 大部分 URL 主要有三个部分：协议、主机名、目录或文件名。

    • 主机名后面可能包含端口号，路径后面可能有查询，最后是锚。

# 3.Web 发布

# 3.1 web 特性

超文本：非线性信息阅读文本，可以视为链接。web 就是一种超大型的超文本信息共享网点。公司通常会将公司信息放在安全私有的内联网（使用 internet 协议但是不对外公开）中，确保信息不外流。而 web 核心的概念就是跨平台的特性，但是现在由于部分功能例如 flash，跨平台会有损失。

每个网页与网页元素都会有自己的唯一地址，称为 “统一资源定位符 URL”。

web 上可以集成各种部件与功能，例如论坛，游戏等。web 的发布与运维需要在 web 服务器上进行。

需要注意：不同的浏览器可能会对网页有不同的显示，所以尽量不要只针对一个网页做针对开发。

# 3.2 网站剖析

网站：一个或多个网页，使用特定含义的方式组织起来。

web 服务器：internet 或内联网中的计算机，在收到浏览器请求时候提供网页或其他文件。

网页：网站上的单个文档。由 HTML 文档与内容组成。现在很多网站并不是由网页组成的，而是由应用程序组成。

主页：网站的入口页面。但是用户可以从其他子网页进入，所以用户不一定非要访问主页。

开发者工具：浏览器中的一个面板，Windows 快捷键为 ctrl+shift+I

# 3.3 网站规划

确定自己要上传什么

确定如何在网页之间分配内容，并且制定在网页之间导航的方案。即绘制线框图。

在绘制框线图时候，需要注意的问题：如何分配网页的具体主题？网页之间如何导航跳转？需要提供哪些额外的导航方式？（例如索引，版权信息等）要在主页放置哪些内容？如何让用户理解自己的当前位置？清楚自己的目标是什么？

# 3.3 网站托管

获取一个 web 托管账户，以将网站上传至服务器。主要有两种方式：

  • 使用内容管理应用程序：例如 Weebly，Wix，Squarespeace

  • 设置 web 托管，例如 DreamHost，Pair.com

设置托管服务的步骤：
1. 注册一个域名
2. 选择一个托管公司购买托管服务，如果不需要通过 internet 访问可以不要购买托管服务。
3. 将注册的域名与新创建的网站关联起来。
4. 开始上传内容。可以利用 FTP，SCP 等文件传输工具。

# 二、创建网页

<**!DOCTYPE>** 用于告诉有效性验证器与浏览器，网页是哪种规范编写，此处为 HTML5。XHTML 和 HTML5 都要求必须有 DOCTYPE 标签。其后面将紧跟着 < html > 标签。

浏览器在读取 HTML 文件时候，将会忽略所有的空格，制表符与换行符。好处是可以将标签任意放置。

id 属性可以唯一的标识该页面元素。在同一个标签中，属性可以有多个。

class 是一个全局属性，用于建立元素编组，方便 CSS 中引用调整。

在 HTML 中，部分结束标签是可以选择的。在 XHTML 中要求所有都必须有结束标签。

title 标签不允许包含其他标签，每个页面只能有一个名称。为了用户能理解页面的作用，请给定简单易懂的名称。名称理论上不限长度，但是浏览器显示有限，可能会截断。

注意！不要只为了粗体效果使用 <h1> 等标题，这将会影响网页的结构和检索效果。

在建立页面索引时，搜索引擎获取每个页面的名称，然后显示搜索结果。所以 title 标签将会影像搜索结果。

URL 代表着是统一资源定位符（Uniform Resource Locator）。URL 无非就是一个给定的独特资源在 Web 上的地址。理论上说，每个有效的 URL 都指向一个唯一的资源。这个资源可以是一个 HTML 页面，一个 CSS 文档，一幅图像，等等。而在实际中，也有一些例外，最常见的情况就是一个 URL 指向了不存在的或是被移动过的资源。由于通过 URL 呈现的资源和 URL 本身由 Web 服务器处理，因此 web 服务器的拥有者需要认真地维护资源以及与它关联的 URL。

# 相对路径与绝对路径

相对路径名必须使用 UNIX 式路径，使用（/）分隔目录名，使用（..）表示当前目录的父目录。例如：“../files/file.html” 表示 file.html 位于当前目录的父目录底下的 files 目录中。

绝对路径表示从顶级目录出发，需要经过哪些目录到达文件。如果使用本地磁盘发布网页，则顶级目录指系统盘。如果使用 web 服务器发布文件，顶级目录即 web 服务器提供的所有文件所指的目录，这种目录称为文档根目录。例如：“/home/lemay/file.html”

通常来说，如果链接的页面属于同一个集合，使用相对链接。如果它们属于不同的集合，使用绝对路径。好处是，在修改目录位置时，不会出现错误，无需修改。

# 5. 字符实体

因为在 HTML 中使用了部分特殊字符，所以在需要书写该字符时，要用到字符实体。注意实体的书写对于大小写是敏感的。例如用 “&1t” 表示小于号（<）；

需要注意的是，空格是 HTML 中最普通的字符实体，通常情况下，HTML 会删去文档中的空格与回车，如果需要增加空格，可以使用 & nbsp；

# 6.div 描述页面布局

div 本身是一个容器元素，其中包含了五种布局的元素：header，nav，article，aside，footer

# article

代表文档、页面或应用程序中的一个独立的完整的可以被外部引用的内容。例如一个论坛帖子，一段用户评论或

独立的插件。在其中可以再次嵌套 <header> 或者 footer，为文档内自己的私有部分，删去不影响显示效果。

# section

section 用于定义节，一般用于成节的内容，，其中的内容可以单独存储到数据库中，强调分区与分块，与 article 不同。

需要注意的是：

  • section 元素不是用于设置样式的页面容器，如果需要承载内容并且调整样式，div 元素是一个更好的选择

  • 如果 article，aside 或 nav 元素更符合使用场景或者语义描述，就不要使用 section

  • section 内应该包括有标题的定义。

# nav 元素

用于页面导航的链接元素组，其中的链接可以链接到其他位置。会使 HTML 语义表达更加清楚。在实际使用中，将主要的，基本的链接放到 nav 里面就行。

# aside

用于存放非正文的内容，视为页面中的单独部分，可以被删除。例如广告，成组的链接，侧边栏等。

主要有两种使用方法：

  • 包含在 article 内，通常可以用于放置附属信息，例如参考资料，名词解释等。

  • 包含在 article 外，作为页面或者全局站点的附属信息，例如侧边栏，其中可以是友情链接，文章列表，帖子等。

# header

具有引导和导航作用的结构元素，通常用来放置整个页面或页面内的一个内容区域的标题，也可以包括表格，图片等内容。注意 header 可以存在多个，可以为每一个内容区域添加一个 header。

# footer

一般用于上层容器元素的脚注。包括脚注信息，例如作者，相关阅读链接和版权信息等。与 header 相同可以有多个。

# time

time 是 HTML5 的一个语义化元素，主要目的是使得页面语义描述更清楚。使得搜索引擎更好理解网页页面。

使用 time 可以添加计算机可以识别的时间日期，也可以呈现给用户。添加 time 元素，可以添加例如发布时间，事件发生时间等信息，还可以使用其他的技术（例如日历系统）。其主要有两个可选属性：

# datetime

# pubdate

布尔属性，代表最近的父 article 元素内容的发布日期和时间，如果没有找到父 article 元素，则指向整个文档。每一个 article 元素只能有一个带 pubdate 的 time 元素。其余与 datetime 一致

# hgroup 元素

标题组，一般作为 header 标签的子元素。一个内容区域中包括了主标题和至少一个子标题时候才使用 hgroup。

# address 元素

语义元素，地址，用于呈现在文档中的联系信息，例如作者，网站链接，邮箱等。

# main 元素

代表网页中的主要内容。主要内容是指与网页主题或者 web 应用主要功能直接相关的内容。main 包含每个网页所特有的内容，不应该包括导航条、版权信息、网站 logo、公共搜索表单等共同内容。

需要注意，每个网页只能有一个 main 元素，不能把 main 元素放在任何 article、aside、footer、header 或者 nav 内部。

# 7. 图像元素和多媒体元素

# 图像标记 <img>

用于链接到某一图像的位置并且显示。属性有 src，title 等。src 为必须属性，用于指定文件的路径。title 为图片的替代文字，在图片无法显示或者鼠标停留的时候显示。

# 多媒体文件标记 <embed>
