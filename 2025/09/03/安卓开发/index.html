# 一、历史 & 背景

# 1. 什么是安卓？安卓的发展历史是什么？

1969 年贝尔实验室 UNIX 操作系统诞生，是最主要的操作系统，由此延伸出 iOS 和 MacOS

安卓基于 Linux 开发，而 Linux 来源于 miniUnix 操作系统。

AOSP 安卓开源项目

GMS 谷歌移动服务

MD 设计规范（metarial design 规范）

安卓是一个基于 Linux 内核的开放源代码操作系统，主要由美国 Google 公司和开放手机联盟（Open Handset Alliance）领导和开发。

安卓系统主要用于移动设备，如智能手机和平板电脑。它以其开放性和免费性，成为世界上最流行的移动操作系统之一。安卓系统提供了丰富的应用程序接口（API）和软件开发工具包（SDK），支持开发者在各种设备上创建自定义应用程序。其用户界面通常采用拖动式菜单，并提供了许多内置应用程序，如电子邮件客户端、网络浏览器、通讯录、短信、相机、电话等。此外，安卓系统还支持多种个性化选项，使用户可以调整设备的外观和功能。

除了手机和平板电脑，安卓系统的应用范围越来越广，如可穿戴计算机、车载娱乐系统，甚至一些使用英特尔和 AMD 处理器的计算机也开始运行安卓系统。

# 2.kotlin 是什么？

Kotlin 是一种现代化的、静态类型的编程语言，由 JetBrains 公司开发。它最初是为 Java 虚拟机 (JVM) 设计的，可以编译成 Java 字节码，与 Java 代码无缝互操作。此外，Kotlin 还可以被编译成 JavaScript 源代码或使用 LLVM 编译器基础设施编译成本地二进制文件，如机器码或 iOS 应用程序等，使其能够在多平台上运行，包括浏览器和嵌入式系统等非 JVM 环境。

Kotlin 语言旨在提供更简洁、更安全、更具表达力和更高效的编程体验。它在语法上相对于 Java 更为简洁直观，同时提供了空值安全、智能类型转换、扩展函数等一系列功能来增强代码的健壮性和可读性。由于这些优势，Kotlin 被广泛用于各种类型的应用程序开发中，尤其是在 Android 平台的开发中备受欢迎，甚至已经被 Google 正式宣布为 Android 官方支持的开发语言之一。

全面兼容 Java，支持 Android 开发

# 二、发展前沿

# 发展现状

在技术层面，安卓开发正朝着更加模块化、组件化和智能化的方向发展。Jetpack Compose32 作为谷歌推出的现代化 UI 工具包，正在逐渐成为安卓 UI 开发的主流，它通过声明式编程简化了 UI 的开发流程。Kotlin2 语言因其简洁性和安全性，已成为安卓官方推荐的开发语言。此外，AI 技术的融合 4，如 AICore 和 Gemini 模型，为安卓应用带来了更智能的功能，如文本摘要、智能回复等。折叠屏和大屏适配 342 也是当前的技术热点，随着设备形态的多样化，开发者需要关注不同屏幕尺寸和交互方式的适配问题。

对于想要进入安卓开发领域的学习者来说，首先需要掌握 Java 或 Kotlin 编程语言，了解安卓开发的基础概念和框架。随后，可以通过构建简单的应用来实践所学知识。随着经验的积累，可以学习更高级的技术，如 Jetpack Compose、Kotlin Flow、协程等。同时，关注官方文档和社区动态，了解最新的技术趋势和最佳实践。对于已经有一定基础的开发者，可以通过深入研究源码、参与开源项目和阅读技术博客等方式，不断提升自己的技术水平。

# 薪资水平

因为大多数的 Android 开发者，基本都是做的业务层面的开发，每天就是写着简单的界面，套用的被人的框架在工作。所以工作过三、五年后，发现技术没有得到半点提升，对 Framework 底层这一块还只是停留在表面的了解。

随着 Android 开发行业的快速发展，市场需求也在不断提升，导致低端 Android 开发市场就业大环境不好、行业趋势下滑，使得不少初中级的 Android 开发开始失业，找不到工作。

2023 年程序员就业变难，主要有以下几个原因：\n\n1、前几年 Android 行业正处在风口，热度很高，使得越来越多的年轻人进入这个行业，比如跨考计算机、离职培训编程的…\n\n2、大学扩招，自从 2008 年之后，大学毕业生每年都在不断增加，再加上疫情三年积压的大学生存量，导致竞争很激烈。\n\n3、全球经济下行，互联网行业也一样，同时人工智能的发展也会取代一部分人力，公司为了降本增效一定会裁员或减少招聘量。\n\n4、由于 APP 创业风口已过，大批创业公司死亡，导致了 Android 岗位减少。

我国的互联网行业还在持续发展中，在这个即将步入的万物互联时代，各行各业现在都无法跟互联网和软件脱节。\n\n 当然也就离不开程序员，IT 行业对人才的需求依然在不断增加。\n\n 虽然不可能回到最开始会写两行代码就能找到工作的时期，但依然充满着机遇，中高端人才任然供不应求。

做安卓快满五年了，跟楼上一起也给一些代码方面小建议

1. 不要放过任何 AS 提示，包括且不限于命名，警告，废弃方法。代码规范性很重要。尽可能做一个代码洁癖的人。可以再额外引入阿里巴巴代码规范插件。

2. 多写注释，不光方便阅读，也方便搜索。

3. 有层主说到注意一下分包。可以先接触下 APP 架构。mvp，mvvm 都是非常有代表性的，比 mvc 甚至没有架构清晰的多。

4.UI 方面，很多都在推荐 Google 的 material design 设计规范。但其实对于没有接触过设计，UI 的同学来说。引入 qmui 三方库是最简单且美感的做法。

5. 说到了三方库。要多学会找 github 上的轮子。能不能快速用轮子解决问题决定了做开发的下限。

6. 有问题一定要 Google，其次再百度。

7. 安卓官网的开发入门文档是个宝。

加油～～～!

UI 的话可以考虑参考一下 google 最新出的 material you 设计规范，没记错的话原声 java 的库已经支持绝大部分 material you 设计风格的控件了，如果 up 主是用 Kotlin 开发的话可以自己去看一下，我目前在搞 flutter 所以对于 Kotlin 这块不是很了解，但 up 主能独立完成做出来能力还是很强的，我的建议是可以在 UI 方面多打磨一下
